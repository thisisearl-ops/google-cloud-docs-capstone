Overview

Diagnose and resolve permission errors for Gemini Enterprise endpoint calls in GCP. This skill guides you through identifying the caller identity, checking IAM roles, and granting necessary permissions.

Skill.md

Goal: Fix 403 permission error on Gemini Enterprise endpoint

Context: When aiplatform.endpoints.predict permission is denied

Project: recipe-assistant-test-p06

Automated actions:

  • Detect active service account or user identity
  • Query IAM policy for existing roles

Manual confirmations:

  • User confirms which identity to grant permissions to
  • User executes role binding (CLI or Console)
  • User runs: python3 test_endpoint.py to verify fix

Step-by-step Guide

1

Identify caller identity

First, determine which identity is being used to call the endpoint. Confirm whether the request is coming from your user account, a service account, Cloud Run, or another automation context.

2

Check current IAM roles

Once you know the identity, verify which IAM roles it has on the project.

For project recipe-assistant-test-p06:

Bash 
gcloud projects get-iam-policy recipe-assistant-test-p06 \
  --flatten="bindings[].members" \
  --filter="bindings.members:YOUR_EMAIL" \
  --format="table(bindings.role)"

Look for roles/aiplatform.user in the output before changing IAM policy.

3

Grant Vertex AI User role

The roles/aiplatform.user role includes the required permissions for endpoint calls. Grant it to the caller identity only after confirming the target principal.

4

Verify access

After granting the role, test that the endpoint is now accessible. Re-run your endpoint request and confirm the 403 error is resolved.
